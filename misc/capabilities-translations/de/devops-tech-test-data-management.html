Hinweis: Die Verwaltung von Testdaten gehört zu einer Reihe von Funktionen, die zu einer höheren Softwarebereitstellung und Unternehmensleistung führen. Diese Faktoren wurden im Rahmen des "State of DevOps"-Forschungsprogramms des DORA-Teams ermittelt, einer unabhängigen, wissenschaftlich exakten Untersuchung der Praktiken und Faktoren, die zu einer hohen Leistung führen. Weitere Informationen finden Sie in unseren Artikeln zu DevOps.

Automatisierte Tests sind ein wichtiger Bestandteil moderner Softwarebereitstellungspraktiken. Die Möglichkeit zum Ausführen umfassender Einheiten-, Integrations- und Systemtests ist für den Nachweis unerlässlich, dass sich Ihre Anwendung oder Ihr Dienst erwartungsgemäß verhält und sicher in der Produktion bereitgestellt werden kann. Damit Ihre Tests realistische Szenarien validieren, ist es von entscheidender Bedeutung, den Tests realistische Daten zur Verfügung zu stellen.

Testdaten sind wichtig, da sie für alle Arten von Tests in der gesamten Testsuite erforderlich sind, einschließlich manueller und automatisierter Tests. Mit guten Testdaten können Sie gängiges oder wichtiges Nutzerverhalten validieren, auf Grenzfälle testen, Mängel reproduzieren und Fehler simulieren.

Allerdings ist die effektive Nutzung und Verwaltung von Testdaten kein Kinderspiel. Eine übermäßige Abhängigkeit von Daten, die außerhalb des Testumfangs definiert sind, kann Ihre Tests problemanfällig machen und die Kosten für die Pflege der Daten erhöhen. Abhängigkeiten von externen Datenquellen können zu Verzögerungen führen und die Testleistung beeinträchtigen. Das Kopieren von Produktionsdaten birgt Risiken, da sie möglicherweise vertrauliche Informationen enthalten. Damit Sie diese Herausforderungen bewältigen können, müssen Sie Ihre Testdaten umsichtig und strategisch verwalten.

Testdatenverwaltung implementieren

Eine von DevOps Research and Assessment (DORA) durchgeführte Analyse zeigt, dass erfolgreiche Teams die Testdatenverwaltung mit den folgenden Grundprinzipien angehen:

  • Für die Ausführung vollständig automatisierter Testsuiten sind ausreichend Testdaten verfügbar.
  • Testdaten für automatisierte Testsuiten können bei Bedarf abgerufen werden.
  • Testdaten schränken die automatisierten Tests nicht ein, die Teams ausführen können.

Zum Verbessern von Prozessen zur Testdatenverwaltung sollten Sie möglichst jede dieser Bedingungen in allen Entwicklungsteams erfüllen. Diese Praktiken können auch einen positiven Beitrag zu Ihren allgemeinen Testautomatisierungs- und Continuous-Integration-Funktionen leisten.

Häufige Probleme bei der Testdatenverwaltung

Das Verwalten von Testdaten kann schwierig sein. Die folgenden Probleme treten häufig bei der Testdatenverwaltung auf:

  • Übermäßige Abhängigkeit von Daten beim Testen. Insbesondere sollten Einheitentests nicht von testexternen Daten oder Zuständen abhängen.
  • Verwendung einer vollständigen Kopie der Produktionsdatenbank, anstatt relevante oder wichtige Teile zu identifizieren.
  • Kein Maskieren oder Hashing von sensiblen Daten.
  • Abhängigkeit von Daten, die veraltet oder nicht mehr relevant sind.

Möglichkeiten zum Verbessern der Testdatenverwaltung

Die folgenden Praktiken können Ihnen dabei helfen, Testdaten effektiver und effizienter zu nutzen:

  1. Verwenden Sie bevorzugt Einheitentests. Einheitentests sollten voneinander und von allen anderen Teilen des Systems außer dem zu testenden Code unabhängig sein. Einheitentests sollten nicht von externen Daten abhängig sein. Wie in der Testautomatisierungspyramide definiert, sollten Einheitentests den Großteil Ihrer Tests ausmachen. Gut geschriebene Einheitentests, die auf einer gut durchdachten Codebasis ausgeführt werden, sind viel leichter zu sichten und kostengünstiger zu pflegen als Tests auf höherer Ebene. Wenn Sie mehr Bereiche mit Einheitentests abdecken, können Sie die Abhängigkeit von Tests minimieren, die auf höherer Ebene angesiedelt sind und externe Daten nutzen.

  2. Minimieren Sie die Abhängigkeit von Testdaten. Testdaten erfordern sorgfältige und kontinuierliche Pflege. Wenn sich Ihre APIs und Schnittstellen weiterentwickeln, müssen Sie zugehörige Testdaten aktualisieren oder neu erstellen. Dieser Prozess stellt einen Kostenfaktor dar, der sich negativ auf die Teamgeschwindigkeit auswirken kann. Daher empfiehlt es sich, die Menge der Testdaten zu minimieren, die zum Ausführen automatisierter Tests erforderlich sind.

  3. Isolieren Sie die Testdaten. Führen Sie die Tests in klar definierten Umgebungen mit kontrollierten Eingaben und erwarteten Ausgaben aus, die mit tatsächlichen Ausgaben verglichen werden können. Achten Sie darauf, dass die von einem bestimmten Test verwendeten Daten explizit mit diesem Test verknüpft sind und nicht durch andere Tests oder Prozesse geändert werden. Nach Möglichkeit sollten die Tests den erforderlichen Zustand im Rahmen der Einrichtung selbst erstellen und dabei die APIs der Anwendung verwenden. Das Isolieren der Testdaten ist außerdem eine Voraussetzung für die parallele Ausführung von Tests.

  4. Minimieren Sie die Abhängigkeit von Testdaten, die in Datenbanken gespeichert sind. Die Pflege von Testdaten, die in Datenbanken gespeichert sind, kann aus folgenden Gründen besonders schwierig sein:

    1. Schlechte Testisolierung. Datenbanken speichern Daten dauerhaft. Alle Änderungen an den Daten bestehen testübergreifend fort, sofern sie nicht explizit zurückgesetzt werden. Weniger zuverlässige Testeingaben erschweren die Testisolation und können die Parallelisierung verhindern.
    2. Leistungsbeeinträchtigung. Eine schnelle Ausführung ist eine wichtige Voraussetzung für automatisierte Tests. Die Interaktion mit einer Datenbank ist normalerweise langsamer und umständlicher als die Interaktion mit lokal gespeicherten Daten. Bevorzugen Sie daher nach Möglichkeit In-Memory-Datenbanken.

  5. Machen Sie Testdaten leicht verfügbar. Das Ausführen von Tests auf Basis einer Kopie einer vollständigen Produktionsdatenbank ist mit Risiken verbunden. Das Aktualisieren der Daten kann eine echte Herausforderung sein und langsam vonstattengehen. Deshalb sind die Daten möglicherweise nicht auf dem neuesten Stand. Außerdem können Produktionsdaten vertrauliche Informationen enthalten. Identifizieren Sie also stattdessen relevante Datenbereiche, die für die Tests erforderlich sind. Exportieren Sie diese Bereiche regelmäßig und machen Sie sie für Tests leicht verfügbar.

Testdatenverwaltung messen

Im Zuge der Weiterentwicklung Ihrer Testdatenverwaltung ist es wichtig, den Fortschritt anhand der zuvor beschriebenen Grundprinzipien zu messen.

  1. Für die Ausführung vollständig automatisierter Testsuiten sind ausreichend Testdaten verfügbar. Für diese Messung können Sie die Zeit erfassen, die Entwickler und Tester mit dem Verwalten und Bearbeiten von Daten für die Verwendung in Testsuiten verbringen. Sie können dafür auch wahrnehmungsbasierte Messungen (d. h. Umfragen) durchführen und Teams fragen, ob sie adäquate Daten für ihre Arbeit haben oder der Meinung sind, dass dieser Punkt für sie eine Einschränkung darstellt.
  2. Testdaten für automatisierte Testsuiten können bei Bedarf abgerufen werden. Sie können dies als Prozentsatz der verfügbaren Schlüssel-Datasets und als Anzahl der Zugriffs- und Aktualisierungsvorgänge für diese Datasets messen.
  3. Testdaten schränken die automatisierten Tests nicht ein, die Teams ausführen können. Für diese Messung können Sie die Anzahl der automatisierten Tests erfassen, die ausgeführt werden können, ohne dass zusätzliche Testdaten erforderlich sind. Sie können dafür auch wahrnehmungsbasierte Messungen (d. h. Umfragen) durchführen und Teams fragen, ob sie das Gefühl haben, dass ihre automatisierten Testaktivitäten durch Testdaten eingeschränkt werden.
